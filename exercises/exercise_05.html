Exercise 05.0 - Load a model from a file (live coding)

In this exercise the teacher will show how to use the model loader in the library to import .obj files.

Exercise 05.1 - Adding textures to the model

We loaded the object geometry, but a pure white object would not be that useful for our tests.

If you look in the folder where we have the 3D model, you will find that there are 3 textures. This matches the number of submeshes that have been loaded from the file, so we can use one for each material.

We will try to map those textures to the materials:

  1. Configure the ModelLoader to create new materials for each submesh instead of using the reference material for everything. Check the header file if you have trouble finding the option.
  2. Get each material and change the "Color" uniform to a different color each, to be able to identify each submesh.
  3. Add a ColorTexture sampler2D to the fragment shader, as we did last week. Use it to sample the texture and multiply the Color value in FragColor.
  4. Create a new Texture2DLoader. We will only need it here, so it can just be a local variable. The internal format we will use for this texture is RGBA8. You should be able to deduce the format to use.
  5. For each one of the 3 materials, load and assign a texture to the new uniform texture we added to the shader. Can you match the textures to the correct submesh?
  6. As you probably noticed, textures are not matching the texture coordinates properly. This is because some software expects texture vertical origin to be placed at the bottom of the texture, and others at the top. To fix this, the loader has an option to perform a vertical flip on load, that you should enable.

Take some time to move around the object with the camera and get familiar with it.

Exercise 05.2 - Ambient lighting

Now that we have a textured model, it is time to add lighting. We will implement the Blinn-Phong model for a single point light. In this first part, we will just add the ambient reflection term.

If you remember from the slides, ambient reflection can be computed as this:
Rambient = Ia * Ka * color.

Follow these steps to implement it:

  1. Duplicate the vertex and fragment shaders, and rename them to "blinn-phong.vert" and "blinn-phong.frag", and use them for the reference material.
  2. We already have the color (from the texture), now we need to provide the other two as uniforms to "blinn-phong.frag".
  3. For Ia, we will use a vec3 color uniform, with a more descriptive name, like "AmbientColor". This color should be shared by all the materials, and it is not a really a material property. As such, then, we will add it to the list of filtered uniforms.
  4. Create a member variable to store the ambient color and initialize to a dark gray value (like 0.25). You can set the "AmbientColor" uniform directly to the shader in the shader setup function where we set the camera matrices.
  5. However, the amount of ambient reflection received, Ka, is different for each material and will be (automatically) stored as a property. Add a uniform of type float and name "AmbientReflection", and set a value of 1 for it in the reference material.
  6. Create two functions, GetBlinnPhongReflection() and GetAmbientReflection(). Both should return a vec3 value. For now, the implementation of GetBlinnPhongReflection() should just call GetAmbientReflection().
  7. For the implementation of GetAmbientReflection(), return the value of the ambient formula described before. Pass any parameters that you consider relevant to these new functions.
  8. The final value of FragColor should be the result of GetBlinnPhongReflection(), with an alpha of 1.

Try playing with different values of ambient color, and assign different ambient reflection constants to each material.

Exercise 05.3 - Diffuse lighting

Ambient reflection is a cheap, simple way to model indirect lighting. But, to properly visualize volumes, we need at least diffuse reflection.

The formula for diffuse reflection is as follows:
Rdiffuse = Ilight * Kd * color * (N · L).

In order to add it to the "blinn-phong" shaders, you have to:

  1. Add the diffuse reflection constant, Kd, to the shader; as a uniform of type float, "DiffuseReflection", as we did for the ambient. Initialize it in the reference material with a value of 1 as well.
  2. Add a new uniform color, of type vec3, to the shader, with the name of "LightColor". Again, this should not be a property of the material, so we will add it to the filtered uniforms set.
  3. In the ViewerApplication class, add a member variable for the light color. Because tuning the light intensity just with the color can be difficult, add another float member variable for the light intensity. You can initialize both as 1.
  4. Set the light color uniform directly in the shader setup function, next to the AmbientColor. The value of the light color uniform is the product of the light color and light intensity member variables.
  5. Back to the fragment shader, add a new function GetDiffuseReflection() that returns a vec3. Implement the diffuse reflection formula, except the last term (the dot product).
  6. As you would expect, you should ADD the result of GetDiffuseReflection() to GetAmbientReflection() inside GetBlinnPhongReflection().

Everything until now is almost identical to the ambient reflection. The main difference will be the angle between the light direction and the normal. We already have the normal, but we need to find the light vector. For this, we are going to use a point light, which defined by its position and emits light in all directions.

  1. In the fragment shader, add a new vec3 uniform, "LightPosition".
  2. Create a new vec3 member variable for the light position, next to the light color. Initialize it to any position value you want. Add it to the set of filtered uniforms and set the value in the shader setup function
  3. Back in the shader, we need to find the light vector using the position. Substract the world position from the light position and normalize, to create a vector that goes from the fragment to the light. This will be our vector L. Pass this vector to GetBlinnPhongReflection() and from there to GetDiffuseReflection().
  4. Pass the normalized WorldNormal as well.
  5. Finally, in GetDiffuseReflection() use both to compute the cosine of the incident angle (dot product). You should clamp it to use only positive values.

After all these steps, you should have light coming from the direction of the light, depending on the position that you initialized before. Try using different positions, and observe the results.

Exercise 05.4 - Dear IMGUI

Before adding the final step, the specular reflection, to our shader, we are going to add a debug window.

Recompiling every time we need to change the lighting is not very practical, so it would be great if we could change those values in runtime. In this exercise we are introducing the Dear ImGUI library, which is a great tool for debug GUI. We will continue using this library during the course.

The setup is done for you, as it is not directly related to the course, but feel free to investigate and ask questions if there is something you don't understand.

To add a debug window:

  1. Call RenderGUI() from your Render() function, after you draw the model.
  2. Inside RenderGUI(), use ImGui::DragFloat to add a debug control for the light intensity. The parameters are the name of the control (a descriptive name as a string) and a pointer to where the data is stored, in this case, &m_lightIntensity.
  3. Run the program and observe how the light responds dynamically to the changes. If you don't see any changes, make sure that you are multiplying the light intensity with the light color to set the light color uniform.
  4. Add controls for the other light parameters. You can use ColorEdit3 for colors. To access the float pointer of a vector, you can take the address of the first component.
  5. You can also add controls for some material properties, but you will need to create member variables for them, and set the material property during Update().

Now that you have an efficient way of testing values, have some fun with them before moving to the next exercise.

Exercise 05.5 - Specular lighting

The diffuse lighting already gave the scene a completely different look. But the Blinn-Phong model is not finished yet, we are missing the specular reflection.

Back to the slides, we can read that:
Rspecular = Ilight * Ks * (N · H)exp.

We already have most of the information, but we still need some work:

  1. Add the specular reflection constant, Ks, as a uniform SpecularReflection as we did before for the others. You can initialize it to 1.
  2. Add the another float uniform for the specular exponent, SpecularExponent. For the initial value of this material property, you can use 100.
  3. Implement a new function, GetSpecularReflection(), and add it to the others in GetBlinnPhongReflection(). The initial implementation should be the light color multiplied by the SpecularReflection. Notice that this function does NOT use the object color.
  4. Looking at the initial formula, we are only missing H. This is computed as the half vector between the light vector L, which we already have, and the view vector V, that we don't.
  5. We can compute the view vector V in the same way that we did for the light vector, but using the camera position instead of the light position. We already have that variable as a member in our class, so just pass it as a uniform CameraPosition and set it in the shader setup function. Don't forget to also add it to the filtered uniforms, as this is not a material property.
  6. In the main function of the fragment shader, compute the view vector, then the half vector, and pass it to the GetBlinnPhongReflection() function, and then to GetSpecularReflection(). For the half vector you should use this:
    H = normalize(L + V)
  7. Compute the (clamped) dot product with the normal and the half vector, and apply the SpecularExponent with the function pow.
  8. You should at least add a ImGUI control for the SpecularExponent variable, to observe how the exponent affects the specular reflection.

With this last part, we already have our full Blinn-Phong reflection model. We will see more advanced models in a few lectures, but this is a good starting point!

Exercise 05.X - Get creative (optional)

We will continue working with lights in following exercises, but for now you could try:

  • Move the lighting code to the vertex shader to implement the Gouraud model.
  • Add another point light, calling twice to GetBlinnPhongReflection and adding the result.
  • Attenuate the light intensity inversely proportional to the distance.
  • Replace the point light with a directional light. This should be very simple, just passing the direction as a uniform instead of computing it from the position.
  • Replace the point light with a spot light. For a spot light you will need both the position and the direction, and an angle for the cone. To determine if a point is inside the cone, you can compare the dot product of the direction and the difference of positions.
  • Cast shadow from the directional light.
