Exercise 10 - Distance-aided ray-marching

Learning objectives

This week we will use a different way of rendering objects, ray-marching the scene using signed distance fields (SDF):

  • Practice how to use SDF to create geometric shapes.
  • Create complex scenes combining SDF primitives.
  • Refresh transformation matrices and the difference between world space and view space.

Remember that you can find hints in comments in the code, tagged with the label "(todo)" followed by the number of the exercise.

Exercise 10.1 - Control the objects in the scene

The initial scene contains a sphere and a box. All their parameters are hard-coded in the shader. Try to read the code and understand how it works, and what each parameter is doing.

Your first task is to allow controlling the objects from the GUI.

  1. Replace all the constants with uniforms with the same names, so you don't need to modify the code.
  2. Initialize the material properties with the same initial values that were hard-coded.
  3. Create GUI elements for each property. You can get use the function GetUniformDataPointer() to get a pointer directly to the data, instead of creating member variables.
  4. This is not possible for the transformation matrix, if we want to have proper controls. Instead, use the static variables to modify translation and rotation separately, and build the matrix with them. Then, assign the uniform value to the material.

Play with the new settings, changing colors and positions, and observe the scene.

Exercise 10.2 - Welding the objects

As you can see, to combine the SDF of the two objects, we are using the "Union" operation. But, using ray-marching, we can apply other operations that would not be possible with rasterization.

In this exercise we will use "SmoothUnion" to weld the two objects together, with a smooth transition between them.

  1. Replace the "Union" operation with "SmoothUnion". You need to specify a third parameter, the smoothness. Try some small value to start with.
  2. Move the objects next to each other and see how they react.
  3. Add a uniform for the Smoothness.
  4. Create a GUI control for it too.

As soon as the new distance does not match the sphere, it will take the color of the box. But we can ask "SmoothUnion" to also compute the blend value as a 4th parameter.

  1. Create a local variable of type float for the blend factor.
  2. Pass the new variable as an additional parameter to the "SmoothUnion" function. It acts as an "out" parameter, so the initial value is not important.
  3. For the color computation, replace the conditional with a linear interpolation (mix) between the two colors, using the blend factor.

Before jumping to the next exercise, play with all the settings and see how they behave.

Exercise 10.3 - Camera controller

You may have noticed that the camera controller is still enabled in this application. However, if you try to move the camera, nothing happens.

This is because all the coordinates of our objects are assumed to be in view space.

If we want the transformations to be in world space, we need to convert them to view space before setting the properties in the material:

  1. For the box, we already have the transformation matrix computed in the GUI code. Just add a multiplication with the camera view matrix, and it will react to the camera movements. Do you remember in which side (left or right) you have to place the view matrix?.
  2. For the "SphereCenter", we have to stop using GetUniformDataPointer(). Create an static variable for the center and use it in the GUI control. Then, transform it to view space and assign it to the material uniform.

Be careful, the uniforms will only be updated if the GUI is open! This should be handled in the Update function instead, but it is good enough for the exercise.

Exercise 10.X - Get creative (optional)

You can replace "exercise10.glsl" with your own file (you can start with a copy of empty.glsl) and modify the distance function to recreate a complex scene.

Some interesting ideas:

  • Try to reproduce some basic scene combining primitives (a famous painting, a cartoon character, or anything you want).
  • Explore animation adding a time uniform to your SDFs.
  • Implement some 3D fractal function like mandelbulb, mengel or julia set.
  • Try to add lighting to the scene (remember to pass the light parameters in view space).
